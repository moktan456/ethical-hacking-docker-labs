Get started

First time? About 10–15 minutes — most of it Docker installing. Done once.
1

Install the prerequisites

Docker Desktop (macOS/Windows) or Docker Engine + Compose (Linux) — get Docker ↗ — plus Git and Make. Recommended: 8 GB RAM, 20 GB free disk.

Windows users: run everything in WSL2 (Ubuntu) or Git Bash rather than plain PowerShell/CMD — the Makefile targets need a Unix-style shell.

2

Clone the repo

git clone https://github.com/moktan456/ethical-hacking-docker-labs.git
cd ethical-hacking-docker-labs

3

Get the shared Kali image

Fastest path — pull the prebuilt image and tag it locally (no build, skips all apt/network build issues):
docker pull ghcr.io/michael-borck/ethical-base:latest
docker tag ghcr.io/michael-borck/ethical-base:latest ethical-base

Or build it yourself from base.Dockerfile:

  1. Run make build-base once, before any weekly lab.
  2. Verify it built: docker images | grep ethical-base
  1. Run docker build -t ethical-base -f base.Dockerfile . (don't miss the trailing dot)
  2. Verify it built: docker images | grep ethical-base
4

Start a week

  1. make run-week4 (replace 4 with the week number) — or on Windows: cd labs\week4 && docker compose up -d
  2. Enter the attacker container — check each week's card below for its exact name, e.g. docker exec -it week4-attacker bash
  3. When you're done: cd labs/week4 && docker compose down, or make stop-all to stop everything at once.

The twelve-week programme

One lab per week. Click a card for how to run it and its network layout.

Good to know

A few things that apply across the whole course.

🏁 Optional CTF challenges

Most weeks hide two flags — user.txt and root.txt — inside the containers, format flag{...}. Work from that week's ctf-challenge.md; the walkthrough is instructor-only.

🖥 VulnHub practice machines

Each week's README links two extra VulnHub machines (Easy/Medium) for independent practice outside class. See the Kali VM guide ↗ for how to attack them from your own VM.

⚠ Authorised use only

These labs are for educational use within CYB204. Do not point these tools or techniques at any system outside this controlled environment without explicit written authorisation.

🌐 Network layout

Every week runs on its own 10.10.WEEK.0/24 subnet — the attacker is always at .2 (or the week's stated address), targets start at .10.

Part of a wider series

This course adapts the "Ethical Hacking" lab from Michael Borck's Assume-Breach Series — five labs that form one lifecycle, break in → move through → assume compromise → investigate → govern. The rest may be useful across other ADCI units.

🎯 1 · Break in — Ethical Hacking

This course. Recon, scan, and exploit your way onto a target — twelve labs from enumeration to a full De-ICE box.

🔀 2 · Move through — Lateral Movement

Land a foothold, then pivot through a segmented network to reach internal services. Builds directly on Week 9.

Open lab ↗

🛡 3 · Assume compromise — Defensive labs

Eleven labs in defensive security — scanning, crypto, firewalls, packet capture — assuming the attacker is already in.

Open lab ↗

🔍 4 · Investigate — Digital Forensics

Investigate the Cloudcore breach: recover deleted files, analyse memory, follow the exfiltration, write the report. Runs the same way on Mac, Linux and Windows.

Open lab ↗

📋 5 · Govern — Security Audit

Play the consultant: audit a fictional company against the Essential Eight — read policies, interview staff, substantiate every finding.
